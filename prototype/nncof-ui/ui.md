

## message type

- SUBSCRIBE
- UNSUBSCRIBE
- NOTIFY
- SUBSCRIBED
- UNSUBSCRIBED

## message structure
- from
- to
- type
- data
- timestamp

## 기본동작

- 메시지를 수신하면 from 노드에서 to 노드로 화살표를 그린다.
- 화살표는 색깔로 구분한다.
- subscribe, unsubscribe, notify는 from -> to 로 화살표를 그리는 애니매이션 실행 후 사라진다.
- subscribed는 from -> to 으로 화살표를 그리고 유지한다. (화살표위에 subscribed 표시)
- unsubscribed는 from -> to 화살표를 없앤다. (화살표위에 unsubscribed 표시)

## 시뮬레이션

- 백앤드 연동 없이 메모리에 메시지를 저장하고 시뮬레이션 버튼을 누르면 메시지를 하나씩 실행한다. 시뮬레이션이 진행되는 동안에는 메시지를 추가할 수 없다.
- 메시지목록에서 하나씩 클릭해서 개별로 실행할 수 있다.
- 초기화 버튼이 있어서 누르면 메시지가 초기 상태로 된다.

## 노드
- 노드는 마우스로 클릭해서 드래그 할 수 있다.
- 노드를 클릭하면 노드의 상세 정보를 볼 수 있다.
- 노드를 더블 클릭하  면 노드의 상세 정보를 볼 수 있다.
- 노드를 우클릭하면 노드의 상세 정보를 볼 수 있다.

## 화살표
- 화살표는 마우스로 클릭해서 드래그 할 수 있다.
- 화살표를 클릭하면 화살표의 상세 정보를 볼 수 있다.
- 화살표를 더블 클릭하면 화살표의 상세 정보를 볼 수 있다.
- 화살표를 우클릭하면 화살표의 상세 정보를 볼 수 있다.
